## 3.1. Тема и концепция

### Название приложения
**SwimTrack** - это веб-приложение для пловцов, тренеров и любителей плавания.

### Краткое описание
SwimTrack помогает отслеживать прогресс в тренировках, изучать техники плавания, находить подходящие бассейны и готовиться к соревнованиям. Приложение создано пловцом для пловцов: интерфейс интуитивен, а функционал заточен под реальные потребности спортсменов.

### Целевая аудитория
| Группа | Потребности |
|--------|-------------|
| Пловцы-любители | Ведение дневника тренировок, поиск бассейнов рядом |
| Профессиональные пловцы (КМС, МС) | Детальная статистика, анализ прогресса, календарь стартов |
| Тренеры | Контроль подопечных, планирование нагрузок |
| Родители юных пловцов | Отслеживание успехов ребёнка |

### Ключевой функционал
1. Регистрация и личный кабинет с настройками профиля
2. Дневник тренировок: запись дистанции, времени, стиля плавания
3. Каталог бассейнов с фильтрами (город, цена, удобства)
4. Библиотека техник: кроль, брасс, баттерфляй, спина (описание/видео)
5. Календарь соревнований с возможностью добавления своих стартов
6. Поиск по бассейнам, техникам, статьям
7. Админ-панель для модерации контента и управления пользователями

## 3.3. Технологический стек

### Frontend

| Технология | Обоснование выбора |
|------------|--------------------|
| **HTML5** | Базовый стандарт разметки, поддерживается всеми браузерами |
| **CSS3** | Стилизация интерфейса и адаптивный дизайн |
| **Чистый JavaScript** | Интерактивность без фреймворков, что дает повышенную производительность и отсутствие лишних зависимостей |
| **Axios** | HTTP-запросы: автоматический JSON, удобная обработка ошибок |
| **Роутер** | Navigo, он легкий (4кб), синтаксис как Express, работает с чистым JS |
| **State-менеджмент** | Zustand, он реактивный, тоже легкий (1кб),работает без React |

### Backend

| Технология | Обоснование выбора |
|------------|--------------------|
| **Node.js** | Единый язык для фронтенда и бэкенда, огромное сообщество |
| **Express.js** | Минималистичный фреймворк, минимум кода для API |
| **PostgreSQL** | Промышленный стандарт, транзакции, надежность, облачный хостинг |


### Инфраструктура

| Компонент | Выбор | Обоснование выбора |
|-----------|---------|-------------|
| Хостинг фронтенда | **GitHub Pages** | Бесплатно, интегрировано с репозиторием |
| Хостинг бэкенда | **Render** | Бесплатный тариф для Node.js, авто-деплой |
| База данных | **Neon PostgreSQL** | Бесплатный облачный хостинг PostgreSQL |

## 3.4. Визуальное проектирование интерфейса

**Выбран вариант: В — Подробное текстовое описание с разметкой**

### Страница: Главная (/)
**Тип доступа:** публичный  
**Основные блоки:**
1. Шапка: логотип «SwimTrack», меню (Главная, Бассейны, Техники, Войти)
2. Герой-секция: заголовок «Твой прогресс в плавании», подзаголовок, кнопка «Начать»
3. Секция «Популярные бассейны»: 3 карточки (название, город, рейтинг) — данные из `GET /api/pools?popular=true`
4. Секция «Стили плавания»: иконки кроль/брасс/баттерфляй/спина — данные из `GET /api/techniques`
5. Футер: контакты, ссылки на соцсети 
**Динамический контент:** блоки 3 и 4 загружаются через API

---

### Страница: О приложении (/about)
**Тип доступа:** публичный  
**Основные блоки:**
1. Шапка: логотип «SwimTrack», меню (Главная, Бассейны, Техники, Войти)
2. Заголовок: «О проекте SwimTrack»
3. Описание: информация о назначении приложения, его возможностях и преимуществах
4. Раздел «Стили плавания»: 4 карточки (кроль, брасс, баттерфляй, спина) с кратким описанием — данные из `GET /api/techniques`
5. Раздел «Видеоуроки»: встроенные видео с YouTube (обучающие ролики по техникам)
6. Футер: контакты, ссылки на соцсети 
**Динамический контент:** блок 4 (список стилей загружается с API)

---

### Страница: Каталог бассейнов (/pools)
**Тип доступа:** публичный  
**Основные блоки:**
1. Шапка (как на главной)
2. Фильтры: город (выпадающий список), цена (ползунок), удобства (чекбоксы: душ, парковка, тренер)
3. Сетка карточек бассейнов: фото, название, адрес, рейтинг, кнопка «Подробнее»
4. Пагинация: «Назад» / «1 2 3» / «Вперёд»  
**Динамический контент:** карточки загружаются из `GET /api/pools?city=Москва&maxPrice=500`

---

### Страница: Личный кабинет (/profile)
**Тип доступа:** частный (только после входа)  
**Основные блоки:**
1. Шапка + аватар пользователя
2. Приветствие: «Добро пожаловать, [Имя]!»
3. Вкладки: «Мои тренировки» | «Статистика» | «Настройки»
4. Таблица тренировок: дата, дистанция, стиль, время (данные из `GET /api/workouts`)
5. Кнопка «+ Добавить тренировку» + модальное окно с формой  
**Динамический контент:** таблица и статистика подгружаются через API после авторизации

---

### Страница: Аутентификация (/auth/login, /auth/register)
**Тип доступа:** публичный  
**Основные блоки (для регистрации):**
1. Форма: поля «Логин», «Пароль», «Повторите пароль», кнопка «Зарегистрироваться»
2. Ссылка «Уже есть аккаунт? Войти»
3. Блок ошибок: если логин занят или пароли не совпадают (красный текст)  
**Логика:** при успешной регистрации редирект в `/profile`, данные отправляются `POST /api/register`

---

### Страница: Детальная страница бассейна (/pools/:id)
**Тип доступа:** публичный  
**Основные блоки:**
1. Галерея фото бассейна
2. Информация: название, адрес, время работы, цены
3. Карта (статичное изображение или ссылка на карты)
4. Отзывы пользователей (список: имя, текст, дата)
5. Кнопка «Добавить в избранное» (только для авторизованных)  
**Динамический контент:** данные загружаются из `GET /api/pools/:id`, отзывы из `GET /api/pools/:id/reviews`

---

### Страница: Библиотека техник (/techniques)
**Тип доступа:** публичный  
**Основные блоки:**
1. Сетка карточек: кроль, брасс, баттерфляй, спина (иконка + название)
2. При клике на карточку и переход на `/techniques/:id`
3. Фильтр: «Для начинающих» / «Продвинутые»  
**Динамический контент:** список техник из `GET /api/techniques`

---

### Страница: Поиск (/search)
**Тип доступа:** публичный  
**Основные блоки:**
1. Поисковая строка с автодополнением
2. Результаты: вкладки «Бассейны» | «Техники» | «Статьи»
3. При отсутствии результатов: сообщение «Ничего не найдено» + рекомендации  
**Динамический контент:** результаты подгружаются при вводе: `GET /api/search?q=запрос`

---

### Страница: Админ-панель (/admin)
**Тип доступа:** частный (только role='admin')  
**Основные блоки:**
1. Боковое меню: «Пользователи» | «Бассейны» | «Статьи» | «Статистика»
2. Таблица пользователей: ID, логин, роль, кнопки «Редактировать»/«Удалить»
3. Форма добавления нового бассейна
4. Графики: «Регистрации по дням», «Популярные техники»  
**Динамический контент:** все данные через `GET /api/admin/*` с проверкой прав

## 3.5. Схема данных и API

### Основные модели данных

#### Пользователь (User)
Хранит данные для аутентификации и управления доступом

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INTEGER | Уникальный идентификатор (автоинкремент) |
| `login` | TEXT | Уникальный логин пользователя |
| `password` | TEXT | Хэш пароля (не хранится в открытом виде) |
| `role` | TEXT | Роль: `'user'` или `'admin'` |
| `created_at` | DATETIME | Дата регистрации |

#### Бассейн (Pool) основная сущность
Хранит информацию о бассейнах для каталога

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INTEGER | Уникальный идентификатор бассейна |
| `name` | TEXT | Название бассейна |
| `city` | TEXT | Город расположения |
| `address` | TEXT | Полный адрес |
| `price` | INTEGER | Цена за посещение (в рублях) |
| `amenities` | TEXT | Удобства: "душ,парковка,тренер" |
| `rating` | REAL | Средний рейтинг (0.0–5.0) |
| `created_at` | DATETIME | Дата добавления в каталог |

### Ключевые эндпоинты API

#### Аутентификация

| Метод | Эндпоинт | Описание | Тело запроса | Ответ |
|-------|----------|----------|--------------|---------------|
| `POST` | `/api/register` | Регистрация нового пользователя | `{ "login": "user", "password": "123456" }` | `{ "message": "Успех", "userId": 1 }` |
| `POST` | `/api/login` | Вход в аккаунт, выдача токена | `{ "login": "user", "password": "123456" }` | `{ "token": "jwt...", "user": {...} }` |

#### Бассейны (основная сущность)

| Метод | Эндпоинт | Описание | Параметры | Ответ |
|-------|----------|----------|-----------|---------------|
| `GET` | `/api/pools` | Получить список всех бассейнов | `?city=Москва&maxPrice=500` | `[ { "id": 1, "name": "Чайка", ... }, ... ]` |
| `GET` | `/api/pools/:id` | Получить детали одного бассейна | `:id` — номер бассейна в URL | `{ "id": 1, "name": "Чайка", "address": "...", ... }` |
